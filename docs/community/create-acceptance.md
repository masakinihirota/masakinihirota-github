# コミュニティ作成 — 受け入れ基準

このドキュメントはコミュニティ作成フローの最小受け入れ基準を定義します。

1. ユーザーは自分のプロフィールを持っていること（ルートアカウントに紐づく）
2. コミュニティ作成フォームは少なくとも name（必須）と description（任意）を受け取る
3. サーバー側は認証ユーザーかを検証し、ルートアカウントに対する作成権限があるか検証する
4. 成功時、API は { success: true, communityId: <uuid> } を返す
5. 異常系（validation error / unauthorized / conflict）については適切な HTTP ステータスコードとエラーメッセージを返す

最低限の仕様を満たす実装を TDD で進めてください。
