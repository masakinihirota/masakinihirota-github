# プロフィール表示・編集 — 受け入れ基準

この文書はプロフィール表示・編集機能の最小受け入れ基準を定義します。

必須 (MUST)
- ユーザーはプロフィールページで自身の名前・表示名・自己紹介（bio）を閲覧できること。
- ユーザーはプロフィール編集フォームから表示名と自己紹介を更新できること。
- サーバ側では基本的な入力バリデーション（例: 表示名は3〜50文字）が行われること。

望ましい (SHOULD)
- 変更は即時に UI に反映される（楽観的 UI / 再取得）
- 編集エラーはユーザーに分かりやすく表示される

テストで検証する項目（例）
- ProfileView コンポーネントが user.name / user.displayName / user.bio をレンダリングする
- ProfileEdit フォームは空の表示名を拒否する（バリデーション）
